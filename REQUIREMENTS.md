## 一、课题背景

Agent 技术正在快速发展，多智能体协作是当前的重要方向。本课题要求学生动手实现一个 AI 狼人杀系统，完整体验从角色设计、协作机制到工程落地的全过程。

**为什么选狼人杀**：多角色、信息不对称、既要协作又要博弈，很适合检验 Agent Team 的综合能力。

---

## 二、目标

做一个能自主完成一局狼人杀的 Agent Team：

1. **角色 Agent**：支持狼人、预言家、女巫、猎人、村民等常见角色，各自具备对应的行动与策略
2. **对局引擎**：回合流转、夜间行动、发言、投票、胜负判定
3. **信息隔离**：每个 Agent 只能看到自己身份允许的信息
4. **可观测**：对局日志可查看
5. **加分项**：提供一个前端 UI，支持观战或人机混战

---

## 三、核心挑战

### 3.1 基础课题

| 模块 | 要求 | 验证方式 |
|------|------|----------|
| 角色 Agent | 支持狼人、预言家、女巫、猎人、村民，各自有独立决策逻辑 | 各角色行为符合身份设定，策略有差异 |
| 对局引擎 | 回合流转、夜间行动、发言、投票、胜负判定 | 能完整跑完一局标准狼人杀 |
| 信息隔离 | 每个 Agent 只能看到自己身份允许的信息 | 验证狼人看不到查验结果、村民看不到夜间行动 |
| 可观测性 | 对局日志可查看，关键决策可追溯 | 每回合输入/输出/决策均有记录 |
| 前端 UI | 支持观战或人机混战 | 有可操作的前端界面 |

### 3.2 进阶课题（三选一）

| 方向 | 核心问题 | 具体要求 | 验证方式 |
|------|----------|----------|----------|
| **A. 通用 Agent** | Agent 能否根据目标自主生成/修改代码、测试、部署？ | 能读懂自身代码、能改代码并提交、能自动构建测试重启、失败可回滚、过程可审核 | 以"玩一局狼人杀"为目标，系统自主演化出可运行的 Agent Team |
| **B. 评测+复盘** | 如何系统化评价 Agent 决策质量（不只看胜负）？ | 多维评测（发言/投票/技能）、关键决策复盘、反事实推演、结构化报告、Leaderboard | 构造含明显失误的对局，系统能定位失误并给改进建议 |
| **C. 自进化 Agent** | Agent 能否在多局迭代中持续提升胜率？ | 自动"对局→分析→调整→再对局"闭环、策略适应性、A/B 对战验证、版本回溯 | 终局 Agent 与初始 Agent 对战 20 局，胜率显著提升 |

## 六、评分原则

### 总分 100 分

每个课题独立评选：**卓越项目 1 个（奖金 2 万元）**、优秀奖若干（建议 10 个以内）。

核心选拔目标：**Agent 调优和多智能体系统设计能力**。Agent 相关维度合计 **70%**，工程相关维度合计 **30%**。

### 评分细则

| 维度 | 权重 | 考察要点 | 满分档 | 及格档 | 不及格 |
|------|------|----------|--------|--------|--------|
| **单 Agent 能力**（Prompt 工程与决策质量） | 20% | Prompt 设计质量（角色人设、思维链、few-shot 等）；迭代调优过程；各角色策略差异度；决策推理链路可追溯性 | Prompt 精细且有迭代调优痕迹，不同角色行为差异显著；决策有可追溯推理链路，有量化评估手段，能识别并分析 bad case | 各角色有独立 Prompt 和基本合理行为，但 Prompt 粗糙、策略雷同；有日志但缺推理过程记录 | 所有角色共用 Prompt 或行为随机不合理；无法了解决策原因 |
| **多 Agent 协作与系统设计** | 20% | 上下文管理：每个 Agent 的对话历史、公共/私有信息如何维护与传递；技能管理：角色技能（查验、毒药、开枪等）的抽象与调度逻辑设计；Agent 间博弈行为（伪装、欺骗检测、归票、站队等）；多 Agent 通信与协调机制 | 上下文管理清晰（公共/私有信息分离、历史管理合理），技能调度抽象良好（加新角色/技能改动小），Agent 间有明确博弈行为（归票、交叉验证、选择性暴露等） | 上下文有基本管理但不够精细（如历史过长截断粗暴），技能逻辑硬编码耦合度高；Agent 能按规则交互但缺主动博弈 | 无上下文管理（Agent 每轮无记忆或信息混乱），技能逻辑散落无设计；Agent 各自独立决策不参考他人 |
| **工程实现与系统完整度** | 30% | 对局引擎正确性与边界处理；信息隔离实现（无泄露）；前端可视化与交互体验；可观测性（日志/监控）；代码质量与规范；文档与演示材料完整度 | 对局全流程正确，边界处理完善；信息隔离严格经测试无泄露；前端直观好用，非技术人员能看懂；代码清晰、文档/视频齐全 | 核心流程跑通但部分边界出错；信息隔离基本有效；有前端但粗糙；有基本文档能跑起来 | 对局无法跑完或严重规则错误；信息隔离形同虚设；无前端或无文档，评委无法运行 |
| **进阶课题完成度**（三选一） | 30% | 见下方各方向细则 | — | — | — |

#### 进阶 A：通用 Agent（30%）
- **满分**：以"玩狼人杀"为目标，系统自主演化出可运行的 Agent Team，全流程可审核回溯
- **及格**：能修改自身代码并跑通测试，但自主程度有限，需人工介入较多
- **不及格**：无法完成代码自修改或构建测试的闭环

#### 进阶 B：评测+复盘（30%）
- **满分**：构造含明显失误的对局，系统能精准定位失误并给改进建议；Leaderboard 能区分不同模型/Agent 版本能力
- **及格**：有基本多维评分和报告输出，但复盘深度不足或 Leaderboard 不完整
- **不及格**：仅有胜负统计，无多维评测能力

#### 进阶 C：自进化 Agent（30%）
- **满分**：终局 Agent 与初始 Agent 对战 20 局胜率显著提升，进化过程有版本记录可回溯
- **及格**：有进化闭环但胜率提升不明显，或版本管理不完善
- **不及格**：无法完成自动进化闭环，或无任何可衡量的提升

## 七、加分亮点

以下方向做得好可加分（不限于此）：

1. **策略深度** — Agent 有明确的博弈策略设计，不同角色行为差异显著
2. **工程完整度超预期** — 错误处理、并发支持、监控告警等生产级能力
3. **前端体验出色** — 有动画、状态可视化，非技术人员也能看懂对战
4. **可扩展架构** — 加新角色或规则变体改动量小
5. **人机交互** — 真人玩家可加入对局与 Agent 混合博弈
6. **进阶课题有独创性** — 方案有新意且效果可验证

## 八、我们的技术策略

### 进阶方向选择

| 方向 | 可行性 | 判断 |
|------|--------|------|
| A. 通用 Agent | ❌ 不选 | 跟"做狼人杀游戏"不是同一个范畴，偏 devops/self-modifying 系统 |
| **B. 评测+复盘** | ✅ 推荐 | 与 WereWolfPlus/WOLF 参考最契合，可量化、答辩展示好、风险可控 |
| **C. 自进化 Agent** | ✅ 可选 | 需要大量对局时间，但"进化"概念好出彩 |

> **首选 B，备选 C**。B 确定性高，C 如果时间允许可以叠加做。

| 优先级 | 维度 | 策略 |
|--------|------|------|
| P0 | 单 Agent 能力 (20%) | 参考 WereWolfPlus Prompt 模板，每个角色独立精调，加 CoT + few-shot |
| P0 | 多 Agent 协作 (20%) | 上下文管理精细（公共/私有分离），技能调度抽象化 |
| P0 | 工程完整度 (30%) | 对局引擎无 bug，信息隔离严格，前端体验好 |
| P1 | 进阶课题 (30%) | **建议选 B（评测+复盘）**，与我们已有的参考（WereWolfPlus + WOLF）最契合 |

### 选 B（评测+复盘）的理由
1. WereWolfPlus 已经有完整评测框架（IRP/KSR/VSS + 角色 KPI）
2. WOLF 论文有 deception detection benchmark
3. 评委看重"可量化"，评测天然可量化
4. Leaderboard 视觉效果强，答辩展示效果好
5. 相比 A（通用 Agent 技术风险高）和 C（需要大量对局时间），B 更可控
