## LLM Judge 补充说明

- **Critic round** 当前为 stub 实现：三法官评审后，Critic 的交叉验证和 consensus 检查逻辑尚未完整实现
- Judge 输出解析使用 `re.search(r'\{.*\}', raw)` + `json.loads`，对 markdown 代码块格式敏感，计划迁移到 Pydantic 结构化输出
- 三法官串行调用同一 LLM，非真正并行异构
